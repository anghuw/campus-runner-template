# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please report it responsibly.

**Do NOT open a public GitHub issue for security vulnerabilities.**

Instead, please send an email to the repository maintainer with:

- A description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

## Response Timeline

- **Acknowledgment**: within 48 hours
- **Initial assessment**: within 1 week
- **Fix or mitigation**: depends on severity

## Scope

This is currently a **frontend template with mock data**. There is no backend, no authentication, and no real data processing. Security concerns are minimal at this stage.

As the project evolves to include backend services and real user data, this policy will be updated accordingly.

## Supported Versions

| Version | Supported |
|---------|-----------|
| 0.1.x (current) | ✅ |
| < 0.1.0 | ❌ |
